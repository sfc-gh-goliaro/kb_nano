\section*{Manual review (v2, post file-inspection) --- 10 representative HF models}

\noindent
This is a hand-audited mapping of HuggingFace Transformers compute classes to
the \fastkernels{} kernel files that would actually run them. Every kb-nano
file listed below has been opened and read (init + at least one forward) to
confirm it implements what is claimed; this version corrects 4 mappings I had
wrong in v1 due to docstring-only checks.

Each row is one HF compute class; \texttt{rowspan}'d folder names group
classes by model. Classes captured from \texttt{modular\_<arch>.py}
(preferred) or \texttt{modeling\_<arch>.py}; boilerplate
(\texttt{*PreTrainedModel}, \texttt{*Config}, \texttt{*Output}, mixins) skipped.

\textbf{Status legend:} $\bullet$ \fastkernels{} L4 pipeline file exists
in \texttt{tasks/baseline/L4/}; \cmark{} every required kernel exists in
\fastkernels{}, L4 wiring is the only remaining work; \textbf{P} partial;
\xmark{} unsupported.

\textbf{Mapping legend:} a single \texttt{L<n>/file.py} = direct kernel match;
\texttt{\{file1, file2\}} = compose from listed files;
\textit{composes (wiring)} = pure structural class with no own kernel
(its forward is just composition of the kernels in the classes it contains).

\noindent\textbf{Corrections from v1 (each caught only after the user forced
me to actually open the kb-nano files):}
\begin{enumerate}\itemsep0pt
\item \texttt{bert} --- previously claimed BertSelfAttention used a
  ``clip-attention pattern'' decomposed from L1. Wrong; kb-nano has dedicated
  \texttt{L2/encoder\_attention.py:EncoderSelfAttention},
  \texttt{L2/encoder\_mlp.py}, and \texttt{L2/encoder\_embeddings.py}
  specifically for BERT-family encoders.
\item \texttt{whisper} --- previously claimed WhisperAttention decomposed
  from L1. Wrong; kb-nano has \texttt{L2/whisper\_attention.py} with three
  specialized classes (encoder self, decoder self, cross-attention) and
  \texttt{L2/whisper\_mlp.py}.
\item \texttt{bitnet} --- previously claimed BitNetRMSNorm $\to$
  \texttt{L1/rms\_norm.py}. Wrong; kb-nano has dedicated
  \texttt{L1/bitnet\_rms\_norm.py:BitNetRMSNorm}.
\item \texttt{deepseek\_v3} --- previously listed
  \texttt{L1/flash\_mla\_*.py} directly as the attention mapping. Wrong
  level; the correct top-level wrapper is
  \texttt{L2/deepseek\_mla\_attention.py:DeepSeekMLAAttention} which
  itself uses \texttt{L2/mla\_attention\_impl.py:MLAAttention} +
  \texttt{L2/sparse\_attn\_indexer.py} (V3.2 DSA) + L1 sub-components.
\end{enumerate}

\noindent\textbf{Variant-detection rules applied (lessons from earlier passes):}
fused-vs-unfused activations (\texttt{silu}/\texttt{silu\_and\_mul},
\texttt{gelu}/\texttt{gelu\_and\_mul}, \texttt{relu}/\texttt{squared\_relu\_and\_mul});
norm variants (\texttt{rms\_norm}/\texttt{t5\_layer\_norm}/\texttt{bitnet\_rms\_norm});
RoPE variants (basic/YaRN-NeoX/YaRN-DeepSeek/M-RoPE/vision-2D);
attention backends (Llama paged-KV/MLA/encoder-varlen/Whisper-cross);
MoE expert kernels (\texttt{moe\_grouped\_gemm}/\texttt{mxfp4\_moe});
linear variants (\texttt{linear}/\texttt{bitnet\_linear}); two-layer MLP
(\texttt{encoder\_mlp.py}, \texttt{clip\_mlp.py}, \texttt{whisper\_mlp.py})
vs SwiGLU (\texttt{llama\_mlp.py}).

\noindent\textbf{All 10 verdicts:} 6 $\bullet$ (\texttt{llama},
\texttt{whisper}, \texttt{qwen2\_vl}, \texttt{deepseek\_v3},
\texttt{gpt\_oss}, \texttt{bitnet}) and 4 \cmark{} (\texttt{afmoe},
\texttt{bert}, \texttt{t5}, \texttt{clip}). Zero partial / unsupported in
this sample. Headline: every required compute kernel exists in
\fastkernels{}; the 4 \cmark{} models need L4 wiring files but no new
kernels.

\noindent\textbf{Reverse sanity check on these 10 rows:}
65 unique kb-nano files referenced (33 L1, 12 L2, 11 L3, 9 L4) =
14.1\% of the 462-file kb-nano tree, from 10 rows alone. The previous
L1-only audit referenced 55/462 = 11.9\% across all 448 rows. Scaling
this methodology to 448 rows is expected to push the L3 reference rate
from 0\% to $\sim$50\% and L4 from 18\% to $\sim$60\%.
